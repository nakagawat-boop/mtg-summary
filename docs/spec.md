# 製品仕様書: 企業別応募管理タブ

## 概要

営業MTGサマリーダッシュボードに「企業別進捗」タブ（タブ5）を追加する。
各企業への応募プロセス（応募 → 一次面接 → 最終面接 → 内定 → 決定）を入力・管理・可視化できる機能。

## 背景と目的

現在のダッシュボードはCA単位のKPI集計が中心だが、**企業単位での選考進捗**が一覧できない。
企業ごとのファネル（応募→決定）を可視化し、ボトルネック分析と進捗管理を可能にする。

---

## スプリント構成

### Sprint 1: データ基盤とCRUDフォーム

**ゴール**: 企業別の応募データを入力・保存・表示できる最小機能を実装する。

#### 機能要件

1. **タブ追加**
   - 既存タブの末尾に「企業別進捗」タブを追加（タブインデックス: 5）
   - タブの色は `#e16032` (オレンジ寄りの赤) を使用
   - 既存の5タブ（全体サマリー / SC振り返り / CS振り返り / PJ振り返り / その他トピックス）は一切変更しない

2. **入力フォーム**
   - 以下のフィールドを入力できるフォーム:
     - 企業名（テキスト、必須）
     - 担当CA名（セレクト、SC_CA + CS_CA の全員から選択）
     - 応募数（数値、デフォルト0）
     - 一次面接数（数値、デフォルト0）
     - 最終面接数（数値、デフォルト0）
     - 内定数（数値、デフォルト0）
     - 決定数（数値、デフォルト0）
   - 「追加」ボタンでデータを保存

3. **データ永続化**
   - Supabase に新テーブル `company_progress` を作成して保存する
   - **既存の `mtg_summary_meta` テーブルには変更を加えない**
   - 週キー (`week_key`) に紐づけてデータを保存する

4. **一覧テーブル表示**
   - 入力済みデータをテーブルで一覧表示
   - カラム: 企業名 / 担当CA / 応募数 / 一次面接 / 最終面接 / 内定 / 決定 / 操作
   - 操作列に「編集」「削除」ボタン

5. **編集・削除**
   - 編集: 行をクリックまたは編集ボタンでインライン編集モードに入る
   - 削除: 確認ダイアログ表示後に削除

#### 受け入れ基準

- [ ] 「企業別進捗」タブが表示され、クリックで切り替えられる
- [ ] 既存の5タブが変更前と同じ見た目・動作を維持している
- [ ] フォームから企業データを入力し「追加」で保存できる
- [ ] ページリロード後もデータが残っている（Supabase永続化）
- [ ] 週を切り替えると、その週のデータが表示される
- [ ] テーブルに入力済みデータが正しく一覧表示される
- [ ] 既存データを編集して更新できる
- [ ] データを削除できる（確認ダイアログあり）
- [ ] `npm run build` がエラーなく成功する

---

### Sprint 2: 棒グラフによる可視化

**ゴール**: 企業別のファネル進捗を棒グラフで可視化する。

#### 機能要件

1. **ファネル棒グラフ**
   - 企業ごとに応募数→一次面接→最終面接→内定→決定のファネルを積み上げまたはグループ化棒グラフで表示
   - Chart.js (既存依存) を使用
   - 各ステージに異なる色を割り当て

2. **サマリーカード**
   - テーブル上部に集計カード（KPIカード形式）を表示:
     - 登録企業数
     - 合計応募数
     - 合計決定数
     - 全体通過率（決定数 / 応募数）

3. **レイアウト**
   - 上部: サマリーカード（横並び）
   - 中部: ファネル棒グラフ
   - 下部: データテーブル + 入力フォーム

#### 受け入れ基準

- [ ] 企業別のファネル棒グラフが表示される
- [ ] データが0件のとき、空状態メッセージが表示される（グラフは非表示）
- [ ] サマリーカードに正しい集計値が表示される
- [ ] データを追加・編集・削除するとグラフとサマリーがリアルタイムで更新される
- [ ] 既存タブ（タブ0〜4）の表示・動作に影響がない
- [ ] Chart.js のグラフが既存チャート（SCトレンド等）と共存し、描画エラーがない
- [ ] `npm run build` がエラーなく成功する

## スコープ外（今回は実装しない）

- CSV / Excel エクスポート
- 企業マスタの管理（企業名は自由入力）
- 通知・アラート機能
- 他タブ（全体サマリー等）への企業別データの統合表示
- Slack 通知への企業別データの追加

---

## 既存機能への影響制約

1. **既存タブ**: タブ0〜4のインデックス・ラベル・色・コンテンツは一切変更しない
2. **既存データ**: `mtg_summary_meta` テーブルのスキーマ・データを変更しない
3. **既存API**: `/api/sc-data`, `/api/cs-data`, `/api/sc-history`, `/api/cs-history`, `/api/summary-meta`, `/api/cron/slack` を変更しない
4. **Slack通知**: `/api/cron/slack` と `slack-daily.yml` は影響なし
5. **型定義**: 既存の `CaRow`, `WeekData` 等の型は変更しない（新しい型は追加可）
